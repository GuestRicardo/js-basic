Modelo(Boilerplate)

1 passo:

npm install --save-dev @babel/preset-env @babel/core @babel/cli babel-loader webpack webpack-cli regenerator-runtime core-js@2

Após a instalação sera criado o package.json e esta será o iniciador de tudo, se caso precise deinstalar as mesmas configurações em outra maquina é so copiar o arquivo package.json e colar na pasta onde for instalar o trabalho e apos colar o arquivo so sera preciso digitar no terminal npm install

2 passo:

Dentro da pasta onde foi instalado o projeto crie um arquivo com nome de webpack.config.js

Este arquivo sera responsavel por todas as configurações do webpack.

Agora dentro do arquivo criado é preciso importar um modulo path esse modulo é para a manipulação de caminhos

TODO ARQUIVO NO NODE È UM MODULO
E para usar este arquivo é preciso exporta-lo para q posssa ser usado
//o sistema de modulos desse arquivo funciona com js, ele é diferente d padrao do js,
ESTE PADRÃO É ULTILIZADO PELO NODE
const path = require('patch');

//parte exportada
module.exports ={
//conf.do webpack(na parte dev)
mode: 'development'
//arquivo de entrada
entry: './src/index.js'
//arquivo de saida(bundle.js)
output:{
path: path.resolve(__dirname, 'public', 'assets', 'js'),
filename: 'bundle.js',
},
//vinculando o babel
module: {
rules: [{
exclude: /node_modules/,
test: /\.js$/,
use: {
loader: 'babel-loader'
options:{
presets:['@babel/env']
}
},
}]
},
//fazendo o mapeamento
devtool: 'source-map'
}

3 passo:

Agora sera preciso criar uma pasta q se chama src neste local sera salvo todo codigo fonte, é onde será gerado o arquivo do bundle.js pelo webpack

4 passo:

Agora sera preciso criar uma pasta na pasta raiz, com o nome de public neste local estara tudo q sera mostrado para o usuario, com front da operação, onde estara o index.html, a pasta assets com tudo q vem dentro dela etc.

Observação: tudo q for js tera q ser salvo dentro da pasta public/assets/js inclusive o arquivo bundle
