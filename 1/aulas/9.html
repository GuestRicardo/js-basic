FOR

Sintaxe
for (variavel de controle(de onde vai começar a contar); condiçao q vai parar o laço(quando ele vai parar); e o contador){
resultado pos contagem
}

Exemplo:
for (let i =0; i <= 5; i++){
console.log(`contando ate 5= ${i}`);
}

Checando os numeros pares:
for (let i =0; i <= 50; i++){
const par = i % 2 === 0 ? 'par' : 'impar'
console.log(i, par);
}

Percorrendo um array usando FOR:
const frutas = ['maçã','banana','manga',] for (let i =0; i < frutas.length; i++){
console.log(´Indice ${i}´, frutas[i]);
}

FOR in

O laço for...in somente iterage sobre propriedades enumeradas (lê os indices ou chaves dos objetos) . Objetos criados a partir de construtores built-in (arrays e object)

for(let i in frutas){ //o for in lê os indicadores do array(sera exibbido os indices das frutas)
console.log(frutas[i]) // aq sera exibido os indices e o q contem nos indices(as frutas)

FOR of

Este for é para objetos interaveis, sao elementos q contem indices, como arrays, strings, maps ...

String

const nome = 'Luiz Henrique';
for(let i of nome){
console.log(i); // neste exeplo sera retornado o valor ao inves do indice
}

Arrays

const nome = ['Luiz Henrique', 'Jose', 'Lenon'];
for(let i of nome){
console.log(nome[i]); // neste exeplo sera retornado o valor ao inves do indice
}

FOReach

O forEach()método executa uma função fornecida uma vez para cada elemento da matriz. O forEach()método é um método iterativo . Ele chama uma callbackFnfunção fornecida uma vez para cada elemento em uma matriz em ordem crescente de índice. Ao contrário map()de , forEach()sempre retorna undefinede não pode ser encadeado. O caso de uso típico é executar efeitos colaterais no final de uma cadeia.

const nomes = ['jose', 'maria', 'ismael'];
nomes.forEach(function(elemento-do-array)){
console.log(elemento-do-array); sera exibido, todos os elementos, sem os indices

While e Do while

While

WHILE (checa a condição e depois executa o codigo) essa estrutura é muito parecida com as anteriores, os laços for é usado quando se sabe o tamanho dos elementos, sabendo o quanto de vez sera feito a passada nos elementos.

ja o while e do while é numa situação que nao se sabe o tamanho do elemento, para essa situação que se recomenda o uso dos 2

Sintaxe

let variavel de controle = 0;
while(condição){
o que deseja mostrar apos a verificação
variavel de controle sendo incrementando se caso esquecer tera uma contagem infinita
}

Exemplo

let controlador = 0;
while(i <= 5){
console.log(i)
i++
}
